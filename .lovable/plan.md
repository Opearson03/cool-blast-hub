
# Plan: Synchronize Quote Print and Email PDF Templates

## Problem Analysis
The printed quote (using `PrintableEstimate.tsx`) and the emailed PDF attachment (using the `send-estimate-email` edge function) are generated by **two completely different systems** that have diverged:

| Feature | Print (Client-side) | Email PDF (Edge Function) |
|---------|---------------------|---------------------------|
| Scope breakdowns with specs | Yes | No |
| Project summary cards | Yes | No |
| Two-page layout | Yes | No |
| Exclusions on Page 2 | Yes | Basic footer only |
| Rich template styling | Full implementation | Simplified |

## Solution Strategy

Use **client-side PDF generation** to ensure consistency. Instead of maintaining two separate PDF generation systems, we'll:

1. Generate the PDF on the client using `html2canvas` + `jsPDF` from the same `PrintableEstimate` component used for printing
2. Send the generated PDF to the edge function for email delivery
3. Remove the duplicate PDF generation logic from the edge function

This approach ensures the emailed PDF is **pixel-perfect identical** to the printed version.

### Step 1: Install Required Dependencies
Add `html2canvas` and `jspdf` packages for client-side PDF generation.

- Renders the `PrintableEstimate` component to a hidden container
- Uses `html2canvas` to capture it as an image (scale: 2 for quality)
- Uses `jsPDF` to create a multi-page A4 PDF from the image
- Returns the PDF as a base64 string

Update `send-estimate-email/index.ts` to:
- Accept a pre-generated PDF base64 string from the client

## Benefits of This Approach

1. **Single Source of Truth**: Only `PrintableEstimate.tsx` defines the quote layout
2. **Automatic Sync**: Any future template changes automatically apply to both print and email
3. **Reduced Code**: Remove 400+ lines of duplicate PDF generation from edge function
4. **Full Feature Parity**: Scope breakdowns, exclusions, two-page layout all work in email
5. **Consistent Branding**: Templates preview in settings matches actual output

---

## Risk Mitigation

- **Large PDF Size**: Use JPEG compression for canvas images, quality 0.92
- **Slow Generation**: Show loading indicator, typical time 1-2 seconds
- **Cross-Origin Images**: Use `useCORS: true` in html2canvas for logos
- **Edge Function Timeout**: PDF is pre-generated, edge function just sends email (fast)
